Reproducibility definition

The ability to obtain consistent computational results using the same input data, computational steps, methods, code, and conditions of analysis

  • National Academies Reproducibility and Replicability in Science

Benefits of reproducibility


  • Ensures scientific work can be trusted
  • Facilitates collaboration and review
  • Helps prevent misinformation and duplicate work

Reproducibility vs. replicability


Reproducibility is different from replicability, which involves new data collection to confirm findings.

Reproducibility checklist

A reproducible analysis includes:

  • Data
  • Code for cleaning raw data
  • Code and packages for analysis
  • Careful documentation and explanation of the analysis
  • No “by hand” steps. Everything is done via code.

Ideally all this is all tied up in a neat little package and made publicly available. This is a hallmark of “open science.”

R packages

  • Packages: fundamental units of reproducible R code. They include R functions, the documentation that describes how to use them, and sample data

  • As of January 2026, there are over 23,000 R packages available on CRAN (the Comprehensive R Archive Network)

  • We’re going to work with a small but important subset of these (the tidyverse!)

R packages


Packages are installed with the install.packages() function and loaded with the library() function:

install.packages("package_name")
library(package_name)


Packages only need to be installed once. They need to be loaded every time you open a new R session.

Functions


Function: A reusable block of code that performs a specific task.

Functions are often verbs, followed by arguments that specify how the function will be applied in parentheses: 

do_this(to_this)
do_that(to_this, to_that, with_those)

R is an object-oriented computing environment


When you run code in the console, the code is executed but the results aren’t saved anywhere. 

5 * 4
[1] 20

R is an object-oriented computing environment


If you assign the output of your code as an object using <-, the object will be added to your global environment. They aren’t saved permanently to your computer, but they are stored in the memory of your current R session. You can then do calculations with objects and pass them to functions.

x <- 5 * 4

x
[1] 20

Quarto: Literate programming

Quarto

https://quarto.org/

  • Quarto enables R users to write both their code and prose in reproducible computational documents
  • Quarto documents have a .qmd file extension
  • They use simple markdown syntax for text
  • Code goes in “chunks,” narrative goes outside of chunks

Quarto, objects, and the R environment

Note

The environment of your Quarto document is separate from your global environment!

  • When you render a document, Quarto “forgets” what you already have saved in your environment and re-runs your entire document from scratch. Everything needed to run your code needs to be in your quarto doc in order for it to render.

  • Remember this, and expect it to bite you a few times as you’re learning to work with quarto!
